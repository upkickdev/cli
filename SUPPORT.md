# Support

## Bugs

Use the **Bug report** issue form and provide a minimal reproduction.

## Feature requests

Use the **Feature request** issue form. Explain the user problem before proposing an implementation.

## Usage questions

Check the documentation at <https://docs.upkick.dev> and search existing issues first. GitHub Discussions may be used for questions once Discussions are enabled.

## Security

Follow `SECURITY.md`. Never report a vulnerability publicly.

Upkick is maintained on a best-effort basis. Opening an issue does not guarantee a response time or implementation.
